# Firmware Abstraction Facility (FAF)

This library provides an abstraction layer between hardware and application code, helping you write high-performance, decoupled firmware — so you don't have to rewrite the same logic for every different microcontroller.

## Architecture

The architecture is based on the **Hardware Abstraction Layer** (HAL) model, where users can create drivers for specific components following the library's expected interface, and use them in the application through the API each driver exposes.


```
      ┌───────────────────────────┐
      |     Application Layer     │
      │ Main Code + Business Code │
      └────────────┬──────────────┘
                   │
   ┌───────────────┴─────────────────┐
┌──┤        FAF Common Layer         ├──┐
|  │ Provider / Manager / Driver API │  |
|  └─────────────────────────────────┘  |
*---------------------------------------*
|  ┌─────────────────────────────────┐  |
|  |       FAF Driver Internal       |  |
└──┤    Specific Platform Drivers    ├──┘
   └───────────────┬─────────────────┘
                   |
   ┌───────────────┴─────────────────┐
   │         Hardware Layer          │
   │     Arduino / ESP32 / STM32     │
   └─────────────────────────────────┘
```

Drivers are made available through a global list generated by a built-in **Domain-Specific Language** that resolves entirely at compile time, with no need for external scripts or equivalent tooling.

Full documentation is available in the [/docs](./docs/) folder.

## Examples

Currently only the "*hello_faf*" example is available, demonstrating the basic usage of FAF.

## Limitations

Currently only a few drivers for the ESP32 Arduino platform are provided:

* ESP32 Built-in LED
* ESP32 Built-in WiFi Device (`WiFi.h` as backend)
* Generic Basic MQTT Client (`PubSubClient` as backend)

> **NOTE:** The WiFi Driver does not support the **Wi-Fi Access Point (AP)** functionality.

Users can extend the library whenever they need to communicate with new devices, or to create drivers better suited to their needs than the built-in ones.
