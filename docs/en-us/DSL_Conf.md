# DSL

The FAF DSL is a set of macros that allows configuring and registering drivers in the Provider declaratively, without dynamic allocation and with compile-time checks.

The configuration is split into two files: `FAF_Setup.h`, which exposes the public declaration macro, and `FAF_Setup_Internal.h`, which contains the definition macros used in the project's configuration `.c` file.

## DSL API:

### Macros:

- [DECLARE_PROVIDER_SUPPLY](#DECLARE_PROVIDER_SUPPLY)(list_name)
- [DRIVER_LIST_INIT](#DRIVER_LIST_INIT)(list_name, ...)
- [DRIVER_DECLARE_MEMBER](#DRIVER_DECLARE_MEMBER)(className, var)
- [DRIVER_LIST_APPEND](#DRIVER_LIST_APPEND)(list_name, className, var, cfg, ...)
- [PROVIDER_SUPPLY_CREATE](#PROVIDER_SUPPLY_CREATE)(list_name, ...)
- [PROVIDER_SUPPLY_APPEND](#PROVIDER_SUPPLY_APPEND)(list_name, className, var, nameStr)

<br>

## Macros:

## DECLARE_PROVIDER_SUPPLY

```c
DECLARE_PROVIDER_SUPPLY(list_name)
```

- Return: Undefined
- Input: ID of the configuration list in the form of `TEXT`

<br>

The `DECLARE_PROVIDER_SUPPLY` externally declares the `FAF_ProviderConfig` configuration variable generated by `PROVIDER_SUPPLY_CREATE`. It must be used in the header of the project's configuration file so that the Provider can see the driver list.

### Example:

```c
// CBoard_Config.h
DECLARE_PROVIDER_SUPPLY(config)
```

## DRIVER_LIST_INIT

```c
DRIVER_LIST_INIT(list_name, ...)
// ...
DRIVER_LIST_END
```

- Return: Undefined
- Input: ID of the list in the form of `TEXT`, followed by calls to `DRIVER_DECLARE_MEMBER`

<br>

The `DRIVER_LIST_INIT` opens the definition of an anonymous struct that groups all pairs of `Class` and `Instance` of the registered drivers. Each driver is declared inside the block using `DRIVER_DECLARE_MEMBER`. The block must be closed using `DRIVER_LIST_END`.

> **NOTE:** `DRIVER_LIST_INIT`, `DRIVER_DECLARE_MEMBER`, and `DRIVER_LIST_END` work together as a unit — never use one without the others.

### Example:

```c
DRIVER_LIST_INIT(config,
    DRIVER_DECLARE_MEMBER(driverClass, driver)
)
DRIVER_LIST_APPEND(config, driverClass, driver, NULL)
DRIVER_LIST_END
```

## DRIVER_DECLARE_MEMBER

```c
DRIVER_DECLARE_MEMBER(className, var)
```

- Return: Undefined
- Input: Name of the driver class in the form of `TEXT`, and a variable identifier in the form of `TEXT`

<br>

The `DRIVER_DECLARE_MEMBER` declares within the `DRIVER_LIST_INIT` block a pair of members: the `Class` and the `Instance` of the driver. Furthermore, it validates at compile time that the driver has a defined `SIGNATURE` that is non-zero and different from `ANY_SIGNATURE`.

> **NOTE:** It must be used exclusively inside a `DRIVER_LIST_INIT` / `DRIVER_LIST_END` block.

### Example:

```c
DRIVER_LIST_INIT(config,
    DRIVER_DECLARE_MEMBER(driverClass, driver)
)
```

## DRIVER_LIST_APPEND

```c
DRIVER_LIST_APPEND(list_name, className, var, cfg, ...)
```

- Return: Undefined
- Input: List ID, class name, variable identifier, configuration pointer, and optional initializers via `...`

<br>

The `DRIVER_LIST_APPEND` initializes the members declared by `DRIVER_DECLARE_MEMBER`, binding the `Class` to its corresponding `Instance` and applying the driver configuration. It must be called after the `DRIVER_LIST_INIT` / `DRIVER_LIST_END` block for each declared driver.

Pass `NULL` in `cfg` if the driver does not require external configuration.

> **NOTE:** The `var` and `list_name` must be identical to those used in the corresponding `DRIVER_DECLARE_MEMBER` and `DRIVER_LIST_INIT`.

### Example:

```c
DRIVER_LIST_APPEND(config, driverClass, driver, NULL)
```

## PROVIDER_SUPPLY_CREATE

```c
PROVIDER_SUPPLY_CREATE(list_name, ...)
```

- Return: Undefined
- Input: List ID in the form of `TEXT`, followed by calls to `PROVIDER_SUPPLY_APPEND`

<br>

The `PROVIDER_SUPPLY_CREATE` creates the array of descriptors `FAF_DriverDescriptor` and the variable `FAF_ProviderConfig` that will be consumed by the Provider. The `count` field is automatically calculated at compile time from the generated array.

> **NOTE:** The `list_name` must be the same as the one used in `DRIVER_LIST_INIT` and `DECLARE_PROVIDER_SUPPLY`.

### Example:

```c
PROVIDER_SUPPLY_CREATE(config,
    PROVIDER_SUPPLY_APPEND(config, driverClass, driver, "Driver Example")
)
```

## PROVIDER_SUPPLY_APPEND

```c
PROVIDER_SUPPLY_APPEND(list_name, className, var, nameStr)
```

- Return: Undefined
- Input: List ID, class name, variable identifier, and descriptive name of the driver in the form of `STRING`

<br>

The `PROVIDER_SUPPLY_APPEND` generates a `FAF_DriverDescriptor` initializer for a driver, binding the driver pointer, its descriptive name, and its `constructor`. It must be used exclusively inside `PROVIDER_SUPPLY_CREATE`.

### Example:

```c
PROVIDER_SUPPLY_CREATE(config,
    PROVIDER_SUPPLY_APPEND(config, driverClass, driver, "Driver Example")
)
```
